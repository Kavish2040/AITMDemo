Session hijacked

The attacker replayed the stolen ESTSAUTHPERSISTENT cookie from a different device. No password prompt. No MFA challenge. Microsoft's server accepts the session as fully authenticated.

MFA verified the user's identity — not the request origin. That's the AiTM vulnerability. Password rotation won't invalidate the stolen cookie; only revoking the session will.
